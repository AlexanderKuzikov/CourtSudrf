# CONTEXT.md — CourtSudrf

## Архитектурные решения

### 1. Почему не `go_search`

Эндпоинт `go_search` на sudrf.ru защищён JavaScript-челленджем (Qrator/botdetector).
Сервер проверяет наличие HMAC-подписанной куки, которая устанавливается только после
выполнения JS в браузере.

Вместо этого используются два эндпоинта без JS-защиты:

- **`ya_coords`** — возвращает все федеральные суды РФ одним скриптом (2334+ записи).
  Содержит код, название, адрес (не всегда), координаты.

- **`ya_info`** — возвращает детальную HTML-карточку суда по коду (vnkod).
  Содержит: название, код, адрес, телефон, email, сайт.

### 2. Кодировка

Все ответы sudrf.ru в windows-1251. Декодируем через `iconv-lite` (библиотека, т.к.
встроенный `TextDecoder` не всегда доступен в tsx-среде).

### 3. Разделение адресов

Поле адреса на sudrf.ru может содержать несколько адресов через ";".
Первый адрес считается основным, остальные — ПСП.
ПСП-адреса часто содержат пометку "(ПСП ...)".

### 4. Rate limiting

Используем Bottleneck с 1.5с между запросами (по умолчанию).
Это предотвращает блокировку и даёт ~2400 запросов/час.
Один суд = один запрос `ya_info`. 2334 суда = ~58 минут.

### 5. Формат данных

Совместим с CourtHarvest2:
- `data/courts.json` — единый массив
- `data/prefixes/{RRTT}.json` — по префиксам (первые 4 символа кода)

### 6. Поле `name` в CourtRecord

Приоритет: название из `ya_coords` (может быть полнее) → название из `ya_info`.
Если `ya_info` не отвечает, запись создаётся из `ya_coords` (хотя бы код + имя).

### 7. Типизация кодов судов

Код = RRTTNNNN, где:
- RR = регион (01-99)
- TT = тип (RS, OS, VS, GV, OV, AJ, KJ, UD)
- NNNN = номер в регионе

### 8. ESM

Проект на ESM (type: module в package.json).
Все импорты с расширением `.js`.

## Известные баги и ограничения

1. **`ya_coords` не даёт email/телефон** — только код, имя, адрес (не всегда), координаты.
   Полные данные только через `ya_info`.

2. **Некоторые суды могут отсутствовать в `ya_coords`** — это маловероятно, т.к.
   эндпоинт возвращает все суды с карты.

3. **Адреса в `ya_coords` могут быть пустыми** — для новых регионов (ДНР, ЛНР, etc.)
   адреса часто отсутствуют. `ya_info` в этих случаях тоже может не иметь полных данных.

4. **JS-защита** — эндпоинт `go_search` не используется из-за Qrator.
   Альтернативные эндпоинты (`ya_coords`, `ya_info`, `ajax_search`) работают стабильно.

5. **ИНН не доступен** — sudrf.ru не показывает ИНН судов (в отличие от DaData).
   Это поле всегда `null` и должно заполняться из CourtHarvest2 при слиянии.

6. **Commander + parseInt** — в Commander v13 нельзя использовать `parseInt` напрямую
   как parseFn с default-значением (радикс путается). Нужна лямбда `(v) => parseInt(v, 10)`.
